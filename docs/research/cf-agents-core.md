# Cloudflare Agents SDK — Specialist Technical Brief

> Source crawl: `https://developers.cloudflare.com/agents/` and the URLs enumerated in `https://developers.cloudflare.com/agents/llms.txt`. Cited inline. Crawled 2026-04-30.

This brief is a flat reference for someone building reference docs and LLM playbooks ("skills") for the Cloudflare Agents SDK. It is intentionally dense: API shapes, footguns, and concrete numbers, not prose narrative.

The package surface, in one line: `agents` (server: `Agent`, `routeAgentRequest`, `getAgentByName`, decorators, mixins) + `agents/client` + `agents/react` + `agents/mcp` + `agents/observability` + `agents/vite` + `agents/tsconfig`.

## 1. Agent class anatomy

The `Agent` class extends `Server` which extends `DurableObject`. Constructor signature is fixed by the Workers runtime — you cannot add custom parameters (the runtime always supplies `state, env`). Use `onStart(props?)` for init, `props` for one-time-arg passing (https://developers.cloudflare.com/agents/concepts/agent-class/, https://developers.cloudflare.com/agents/api-reference/agents-api/).

### Generic typing

```typescript
import { Agent, callable } from "agents";

type Env = { /* generated by `npx wrangler types env.d.ts --include-runtime false` */ };
type State = { count: number; status: "idle" | "running" | "done" };

export class Counter extends Agent<Env, State> {
  initialState: State = { count: 0, status: "idle" };
  // ...
}
```

`initialState` is class-level but only applied on first access per instance — every Agent **instance** gets its own SQLite-backed state (https://developers.cloudflare.com/agents/api-reference/store-and-sync-state/).

### Core properties

| Property | Type | Notes |
|---|---|---|
| `this.env` | `Env` | bindings + vars + secrets |
| `this.ctx` | `ExecutionContext` | DO context (`waitUntil`, `blockConcurrencyWhile`, `storage`) |
| `this.state` | `State` | persisted, JSON-serializable only |
| `this.sql` | tagged template | embedded SQLite, `this.sql<Row>\`...\`` returns `Row[]` |
| `this.name` | `string` | the instance's facet name (best-effort; surfaced via partyserver) |

(https://developers.cloudflare.com/agents/api-reference/agents-api/, https://developers.cloudflare.com/agents/concepts/agent-class/)

### Lifecycle hooks (full set)

```typescript
class MyAgent extends Agent<Env, State> {
  // Called every time the DO starts up: first creation, eviction wake, hibernation wake.
  // Receives optional initialization props passed via routeAgentRequest({ props })
  // or getAgentByName(env.X, name, { props }).
  async onStart(props?: unknown): Promise<void> {}

  // HTTP requests (non-WebSocket) routed to this instance.
  async onRequest(request: Request): Promise<Response> {}

  // WebSocket handshake. Reject by calling connection.close(code, reason).
  async onConnect(connection: Connection, ctx: ConnectionContext): Promise<void> {}

  // Each WebSocket frame.
  async onMessage(connection: Connection, message: string | ArrayBuffer | ArrayBufferView): Promise<void> {}

  // Two overloads:
  onError(connection: Connection, error: unknown): void;
  onError(error: unknown): void;

  async onClose(connection: Connection, code: number, reason: string, wasClean: boolean): Promise<void> {}

  // Inbound email (when wired through `routeAgentEmail`).
  async onEmail(email: AgentEmail): Promise<void> {}

  // State-changed observer. source is "server" or a Connection.
  onStateChanged(state: State, source: "server" | Connection): void {}

  // Synchronous validator: throw to reject a state change.
  validateStateChange(nextState: State, source: "server" | Connection): void {}

  // Per-connection toggle for whether to send cf_agent_* protocol frames.
  shouldSendProtocolMessages(connection: Connection, ctx: ConnectionContext): boolean { return true; }

  // Mark a connection as readonly at handshake time.
  shouldConnectionBeReadonly(connection: Connection, ctx: ConnectionContext): boolean { return false; }

  // Workflow lifecycle (see §6).
  async onWorkflowProgress(workflowName: string, instanceId: string, progress: unknown): Promise<void> {}
  async onWorkflowComplete(workflowName: string, instanceId: string, result: unknown): Promise<void> {}
  async onWorkflowError(workflowName: string, instanceId: string, error: unknown): Promise<void> {}

  // Durable execution recovery (see §7 long-running).
  async onFiberRecovered(ctx: FiberRecoveryContext): Promise<void> {}
}
```

(https://developers.cloudflare.com/agents/api-reference/agents-api/, https://developers.cloudflare.com/agents/api-reference/websockets/, https://developers.cloudflare.com/agents/api-reference/store-and-sync-state/, https://developers.cloudflare.com/agents/api-reference/run-workflows/, https://developers.cloudflare.com/agents/api-reference/durable-execution/, https://developers.cloudflare.com/agents/api-reference/email/, https://developers.cloudflare.com/agents/api-reference/readonly-connections/).

### Callable methods (RPC over WebSocket)

```typescript
import { Agent, callable, type StreamingResponse } from "agents";

class Counter extends Agent<Env, State> {
  @callable()
  increment(by = 1): number {
    this.setState({ ...this.state, count: this.state.count + by });
    return this.state.count;
  }

  @callable({ description: "Generate text and stream it", streaming: true })
  async generate(stream: StreamingResponse, prompt: string) {
    for await (const chunk of llm(prompt)) stream.send(chunk);
    stream.end();
  }
}
```

`StreamingResponse` API: `send(chunk)`, `end(finalChunk?)`, `error(message)`. Args/returns must be JSON-serializable; functions, Dates, Maps, Sets are not. (https://developers.cloudflare.com/agents/api-reference/callable-methods/)

**Footgun:** TC39-standard decorators only. **Do not** set `experimentalDecorators: true` in tsconfig; the SDK will silently break `@callable()` at runtime. Extend `agents/tsconfig` and use the `agents/vite` plugin (https://developers.cloudflare.com/agents/api-reference/callable-methods/, https://developers.cloudflare.com/agents/api-reference/configuration/).

### Introspection

```typescript
const methods = agent.getCallableMethods(); // Map<string, { description?, streaming? }>
```

(https://developers.cloudflare.com/agents/api-reference/callable-methods/)

### `getCurrentAgent()` — async-context accessor

```typescript
import { getCurrentAgent } from "agents";

function someTool() {
  const { agent, connection, request, email } = getCurrentAgent<MyAgent>();
  // any of these may be undefined depending on entry path
}
```

Availability per entry path: `onRequest` → agent+request; `onConnect/onMessage` → agent+connection (+request only on connect); `onEmail` → agent+email; RPC → agent+connection. Returned values may be undefined; always null-check (https://developers.cloudflare.com/agents/api-reference/get-current-agent/).

## 2. State management

### `setState` / `initialState`

```typescript
this.setState({ ...this.state, count: this.state.count + 1 });
```

`setState()` performs three steps in order: (1) write to SQLite, (2) broadcast `cf_agent_state` to connected clients (skipping any where `shouldSendProtocolMessages` returned false), (3) call `onStateChanged()` best-effort after broadcast (https://developers.cloudflare.com/agents/api-reference/store-and-sync-state/).

State must be JSON-serializable. **Functions, Dates, Maps, Sets do not serialize** — use ISO strings for dates (https://developers.cloudflare.com/agents/api-reference/store-and-sync-state/).

### Validation hook

```typescript
validateStateChange(nextState: State, source: "server" | Connection) {
  if (nextState.score < 0) throw new Error("score cannot be negative");
  if (this.state.status === "finished" && nextState.status !== "finished") {
    throw new Error("cannot restart a finished game");
  }
}
```

Synchronous, runs before persist+broadcast. `onStateChanged()` is a notification hook only — do not validate there (https://developers.cloudflare.com/agents/api-reference/store-and-sync-state/).

### Per-instance vs per-class

Each Agent **instance** has its own SQLite DB (state + your own tables). `initialState` is a class-level default used **only when no state exists in storage**. There is no shared state across instances — coordinate via a singleton-named instance (e.g. `name: "default"`) if you need it (https://developers.cloudflare.com/agents/concepts/agent-class/, https://developers.cloudflare.com/agents/api-reference/store-and-sync-state/).

### SQL tagged template

```typescript
this.sql`CREATE TABLE IF NOT EXISTS notes (id TEXT PRIMARY KEY, body TEXT)`;
this.sql`INSERT INTO notes (id, body) VALUES (${id}, ${body})`;
const [row] = this.sql<{ id: string; body: string }>`SELECT * FROM notes WHERE id = ${id}`;
```

Tagged-template values become bound parameters (no string injection). Returns rows as a typed array. The DB is a per-instance SQLite database hosted in the Durable Object's storage; state-size cap is documented as 1 GB per Agent instance (https://developers.cloudflare.com/agents/api-reference/rag/, https://developers.cloudflare.com/agents/platform/limits/).

### Schema migrations

Wrangler-level: SQLite-class migrations via `migrations[]` (see §Configuration). For schema changes inside the DB, run idempotent `CREATE TABLE IF NOT EXISTS …` / `ALTER TABLE` in `onStart()`.

### Workflow → Agent state writes

From inside a Workflow step (durable):

```typescript
await step.updateAgentState(state);     // replace
await step.mergeAgentState(partial);    // shallow merge
await step.resetAgentState();           // back to initialState
```

These broadcast to clients automatically (https://developers.cloudflare.com/agents/api-reference/store-and-sync-state/, https://developers.cloudflare.com/agents/api-reference/run-workflows/).

### Protocol messages

Three core JSON frames sent on connect, broadcast on change (https://developers.cloudflare.com/agents/api-reference/protocol-messages/):

- `cf_agent_identity` — agent class name and instance name
- `cf_agent_state` — current state
- `cf_agent_mcp_servers` — connected MCP server list

Suppress per-connection via `shouldSendProtocolMessages()`. Disable identity altogether with `static options = { sendIdentityOnConnect: false }` (note: `agent.identified` then stays `false`, `agent.ready` never resolves) (https://developers.cloudflare.com/agents/api-reference/routing/).

### Client-side state errors

Server sends `cf_agent_state_error` on rejected client `setState()` (e.g. readonly connection). React clients handle via `onStateUpdateError` callback (https://developers.cloudflare.com/agents/api-reference/readonly-connections/).

## 3. Calling agents (server side)

### `routeAgentRequest(request, env, options?)`

```typescript
import { routeAgentRequest } from "agents";

export default {
  async fetch(request: Request, env: Env, ctx: ExecutionContext) {
    const res = await routeAgentRequest(request, env, {
      cors: true,
      props: { userId: "abc" },
      locationHint: "enam",        // wnam, enam, sam, weur, eeur, apac, oc, afr, me
      jurisdiction: "eu",          // or "fedramp"
      onBeforeConnect: async (req) => {
        if (!verify(req)) return new Response("Unauthorized", { status: 401 });
      },
      onBeforeRequest: async (req) => {},
    });
    return res ?? new Response("Not found", { status: 404 });
  },
};
```

Returns `Promise<Response | undefined>` (undefined when URL doesn't match the agent prefix). URL pattern: `/agents/{agent-class-kebab}/{instance-name}/{sub-path}`. Class names are auto kebab-cased: `ChatRoom` → `/agents/chat-room/...` (https://developers.cloudflare.com/agents/api-reference/calling-agents/, https://developers.cloudflare.com/agents/api-reference/routing/).

**Footgun:** for WebSocket upgrades, **do not modify the response** returned by `routeAgentRequest()` — pass it through as-is or the upgrade fails (https://developers.cloudflare.com/agents/api-reference/calling-agents/).

### `getAgentByName(namespace, name, options?)`

```typescript
import { getAgentByName } from "agents";

const agent = await getAgentByName(env.Counter, "global", {
  locationHint: "enam",
  jurisdiction: "eu",
  props: { userId: "abc" },
});
const n = await agent.increment();    // typed RPC, no @callable required server-to-server
return agent.fetch(request);          // forward an HTTP request
```

Returns a typed `DurableObjectStub<T>`. Naming is by string; same name → same instance forever (https://developers.cloudflare.com/agents/api-reference/calling-agents/, https://developers.cloudflare.com/agents/concepts/agent-class/).

### `routeAgentEmail(email, env, { resolver, onNoRoute })`

```typescript
import {
  routeAgentEmail,
  createAddressBasedEmailResolver,
  createSecureReplyEmailResolver,
  createCatchAllEmailResolver,
} from "agents";

export default {
  async email(message: ForwardableEmailMessage, env: Env) {
    await routeAgentEmail(message, env, {
      resolver: createAddressBasedEmailResolver("EmailAgent"),
      onNoRoute: (email) => email.setReject("Unknown recipient"),
    });
  },
};
```

`AgentEmail` exposes `from`, `to`, `headers`, `rawSize`, `getRaw()`, `reply()`, `forward()`, `setReject()`. Use `postal-mime` to parse. For replies, sign outbound with `secret` to enable `createSecureReplyEmailResolver` (HMAC-validated `X-Agent-Name`/`X-Agent-Id` headers) (https://developers.cloudflare.com/agents/api-reference/email/).

### Naming patterns

- Per-user: `name: \`user-${userId}\`` — isolated state per user.
- Shared room: `name: roomId` — many connections share one DO.
- Singleton: `name: "default"`.

(https://developers.cloudflare.com/agents/api-reference/calling-agents/, https://developers.cloudflare.com/agents/api-reference/routing/).

### Custom routes (`basePath`)

```typescript
// client
useAgent({ agent: "UserAgent", basePath: "user" });

// server
if (url.pathname.startsWith("/user/")) {
  const session = await getSession(request);
  const agent = await getAgentByName(env.UserAgent, session.userId);
  return agent.fetch(request);
}
```

When using `basePath`, identity is server-determined; React clients should provide `onIdentity` and `onIdentityChange` to handle reconnect-time identity flips (e.g. session expiry) (https://developers.cloudflare.com/agents/api-reference/routing/, https://developers.cloudflare.com/agents/api-reference/calling-agents/).

## 4. Client side

### `agents/client` — `AgentClient`

```typescript
import { AgentClient } from "agents/client";

const client = new AgentClient({
  agent: "ChatAgent",
  name: "room-123",
  host: "your-worker.your-subdomain.workers.dev",
  path: undefined,
  query: { token: "..." },
  onStateUpdate: (state, source) => {},  // source: "server" | "client"
});

await client.call("sendMessage", ["Hi"]);
await client.stub.sendMessage("Hi");     // typed proxy
client.setState({ ... });
client.send(rawData);
client.reconnect();
client.close();

client.addEventListener("open" | "close" | "error" | "message", handler);
```

(https://developers.cloudflare.com/agents/api-reference/client-sdk/)

### `agents/client` — `agentFetch`

```typescript
import { agentFetch } from "agents/client";

const r = await agentFetch(
  { agent: "DataAgent", name: "i-1", host: "my-worker.workers.dev" },
  { method: "POST", headers: {"Content-Type": "application/json"}, body: JSON.stringify({}) }
);
```

For one-shot HTTP — no WebSocket (https://developers.cloudflare.com/agents/api-reference/client-sdk/).

### `agents/react` — `useAgent`

```typescript
import { useAgent } from "agents/react";
import type { MyAgent } from "../server";

const agent = useAgent<MyAgent>({
  agent: "MyAgent",
  name: "default",
  host,                                 // defaults to current origin
  path,
  // async query for auth tokens — cached and re-fetched on deps change, ttl, disconnect, remount
  query: async () => ({ token: await getToken() }),
  queryDeps: [userId],
  cacheTtl: 60_000,                     // default 5 min
  onStateUpdate: (state, source) => {},
  onMcpUpdate: (mcp) => {},
  onOpen: () => {},
  onClose: () => {},
  onError: (e) => {},
  onMessage: (e) => {},
  onIdentity: (name, agentType) => {},
  onIdentityChange: (name, agentType) => {},
  onStateUpdateError: (err) => {},
});

agent.state;                            // undefined until first cf_agent_state
agent.setState({...});
await agent.stub.someMethod(args);      // typed
await agent.call("someMethod", [args], { timeout: 5000, stream: { onChunk, onDone, onError } });
agent.reconnect();
agent.close();
```

Reconnection is automatic with exponential backoff. Pending RPC calls reject with `"Connection closed"` on disconnect; await `agent.ready` to retry idempotent ones (https://developers.cloudflare.com/agents/api-reference/client-sdk/, https://developers.cloudflare.com/agents/api-reference/callable-methods/).

### Client → server message envelopes

Documented frame names (https://developers.cloudflare.com/agents/api-reference/protocol-messages/, https://developers.cloudflare.com/agents/api-reference/chat-agents/):

- `cf_agent_identity`, `cf_agent_state`, `cf_agent_mcp_servers` — server-pushed protocol frames
- `cf_agent_state_error` — server → client when a client-driven state update is rejected
- `CF_AGENT_USE_CHAT_REQUEST`, `CF_AGENT_USE_CHAT_RESPONSE`, `CF_AGENT_CHAT_MESSAGES`, `CF_AGENT_CHAT_CLEAR`, `CF_AGENT_CHAT_REQUEST_CANCEL` — chat envelopes used by `useAgentChat`/`AIChatAgent`

State sync flow when client calls `setState`: client → `cf_agent_use_state` → server validates → persists → re-broadcasts `cf_agent_state` to all (including originator with `source: "client"` semantics) → triggers `onStateChanged`.

## 5. Scheduling

### Three overloads

```typescript
// delay (seconds)
await this.schedule(30, "sendReminder", { message: "..." });

// absolute Date
await this.schedule(new Date("2025-02-01T09:00:00Z"), "sendReminder", { /* payload */ });

// cron expression
await this.schedule("0 8 * * *", "dailyDigest", { userId });
```

Signature: `async schedule<T>(when: number | Date | string, callback: keyof this, payload?: T, options?: { retry?: RetryOptions; idempotent?: boolean }): Promise<Schedule<T>>`.

The callback parameter is a method name (string) on the Agent class. **If the method does not exist, the scheduled execution throws.** (https://developers.cloudflare.com/agents/api-reference/schedule-tasks/)

### Cron syntax

Standard 5-field: `minute hour day month weekday`. Supports `*`, `,`, `-`, `/`. Examples:

```
*/5 * * * *      every 5 min
0 8 * * 1-5      weekdays at 8am
0 0 1 * *        first of month
```

### Sub-minute precision

`scheduleEvery(intervalSeconds, callback, payload?)` — supports arbitrary intervals (e.g. 90 seconds) that don't fit cron. **If a callback takes longer than the interval, the next execution is skipped.** (https://developers.cloudflare.com/agents/api-reference/schedule-tasks/)

### Querying & cancelling

```typescript
this.getSchedules();                                                 // all
this.getSchedules({ type: "cron" });
this.getSchedules({ timeRange: { start: new Date(), end: ... } });
this.getSchedules({ id: "abc123" });

const ok = await this.cancelSchedule(scheduleId);   // boolean
```

Schedule object:

```typescript
type Schedule<T> = {
  id: string;
  callback: string;
  payload: T;
  time: number; // unix seconds
} & (
  | { type: "scheduled" }
  | { type: "delayed"; delayInSeconds: number }
  | { type: "cron"; cron: string }
  | { type: "interval"; intervalSeconds: number }
);
```

(https://developers.cloudflare.com/agents/api-reference/schedule-tasks/)

### Idempotency

- Cron schedules are idempotent **by default** (same cron + callback + payload returns the existing schedule).
- Delayed/Date schedules are not — pass `{ idempotent: true }` to dedupe by callback + payload (useful when seeding from `onStart()` to avoid duplicates after restart).

### Alarms

Internally backed by Durable Object alarms; rows live in SQLite. Schedules survive hibernation and trigger DO wakes (https://developers.cloudflare.com/agents/api-reference/schedule-tasks/, https://developers.cloudflare.com/agents/concepts/long-running-agents/).

### Sub-agent caveat

Inside a sub-agent, `schedule()`, `cancelSchedule()`, and `keepAlive()` **throw**. Schedule from the parent and have it dispatch to the child via RPC (https://developers.cloudflare.com/agents/api-reference/sub-agents/).

## 6. Workflows from agents

Use Agents+Workflows for multi-step, retry-on-failure jobs >30s; pure Agents for chat and quick LLM calls (https://developers.cloudflare.com/agents/concepts/workflows/).

### Starting a workflow

```typescript
const instanceId = await this.runWorkflow("PROCESSING_WORKFLOW", { taskId, data }, {
  id?: string,                  // custom; auto-generated otherwise
  metadata?: object,            // queryable, not sent to workflow
  agentBinding?: string,        // auto-detected
});
```

Returns the workflow instance ID (https://developers.cloudflare.com/agents/api-reference/run-workflows/).

### Bidirectional messaging

Workflow → Agent (durable, via `step`):

```typescript
await step.updateAgentState({ currentTask: { id, status: "processing" } });
await step.mergeAgentState({ progress: 0.75 });
```

Agent → Workflow:

```typescript
await this.sendWorkflowEvent("MY_WORKFLOW", instanceId, {
  type: "custom-event",
  payload: { action: "proceed" },
});
await this.approveWorkflow(instanceId, { reason: "ok", metadata: { approvedBy } });
```

### Lifecycle hooks on Agent

```typescript
async onWorkflowProgress(name, instanceId, progress) {}
async onWorkflowComplete(name, instanceId, result) {}
async onWorkflowError(name, instanceId, error) {}
```

### Idempotency model

- `step.do(name, fn)` — **guaranteed exactly once** across retries (durable).
- Non-`step` agent RPC calls (e.g. `this.reportProgress(...)`) **may execute multiple times** on retry; treat as best-effort.
- `step.updateAgentState`, `step.mergeAgentState`, `step.sendWorkflowEvent` — durable.

### Limits

- 30 minutes max per step
- 10 MB max workflow state
- Workflows **cannot open WebSockets**; use Agent's `broadcast()` to fan out to clients
- When renaming a workflow binding in `wrangler.jsonc`, call `migrateWorkflowBinding()` to preserve tracking records

(https://developers.cloudflare.com/agents/api-reference/run-workflows/)

## 7. WebSockets

Lifecycle hooks: §1. WebSocket Hibernation API is on **by default** (https://developers.cloudflare.com/agents/api-reference/websockets/).

### Connection object

```typescript
interface Connection {
  id: string;
  uri: string | null;
  state: any;
  setState(state: any): void;
  send(message: string | ArrayBuffer | ArrayBufferView): void;
  close(code?: number, reason?: string): void;
  readonly tags: string[];        // max 9, ≤256 chars each
  server: string;                 // agent instance name
}
```

### Per-connection state

```typescript
async onConnect(connection, ctx) {
  const token = new URL(ctx.request.url).searchParams.get("token");
  if (!token) { connection.close(4001, "Unauthorized"); return; }
  connection.setState({ userId, role: "user", joinedAt: Date.now() });
}
```

`connection.state` persists across hibernation (https://developers.cloudflare.com/agents/api-reference/websockets/).

### Hibernation persistence

Persists across hibernation: `this.state`, `connection.state`, SQLite, connection metadata, scheduled tasks.

Lost on hibernation: in-memory class fields, timers (`setTimeout`/`setInterval`), in-flight promises, local closures, and any closure captured in `runFiber()` (must use `onFiberRecovered` for restart logic).

### Disabling hibernation

```typescript
class AlwaysOnAgent extends Agent {
  static options = { hibernate: false };
}
```

### Broadcasting

```typescript
this.broadcast(JSON.stringify({ from: connection.id, message }));
this.broadcast(JSON.stringify({ type: "user-typing" }), [connection.id]); // exclude
```

### Tags + selective broadcast

```typescript
getConnectionTags(connection, ctx) {
  return role === "admin" ? ["admin"] : [];
}

for (const conn of this.getConnections("admin")) conn.send(msg);
```

### Connection management

`getConnections(tag?)`, `getConnection(id)`, `isConnectionReadonly(c)`, `isConnectionProtocolEnabled(c)`.

### Notable absences in docs

The websockets page does **not** specify a max message size, ping/pong details, or a `broadcastToClients()` method (only `broadcast()`); it does not document `serializeAttachment`/`deserializeAttachment`. For the Workers runtime defaults, refer to Workers limits docs (https://developers.cloudflare.com/agents/platform/limits/).

## 8. HTTP + SSE

`onRequest` for non-WebSocket HTTP. Use SSE for one-way streaming from server (LLM token streams), WebSockets for bidirectional/interactive (https://developers.cloudflare.com/agents/api-reference/http-sse/).

### Manual SSE

```typescript
async onRequest(request: Request) {
  const stream = new ReadableStream({
    async start(controller) {
      const enc = new TextEncoder();
      controller.enqueue(enc.encode("data: hello\n\n"));
      controller.close();
    },
  });
  return new Response(stream, {
    headers: { "Content-Type": "text/event-stream", "Cache-Control": "no-cache" },
  });
}
```

Format is exactly `data: <line>\n\n`. No timeout limit on Workers SSE responses. Use `Last-Event-ID` for resume (https://developers.cloudflare.com/agents/api-reference/http-sse/).

### AI SDK SSE shortcut

```typescript
import { streamText } from "ai";
const result = streamText({ model, messages });
return result.toTextStreamResponse(); // or toUIMessageStreamResponse() for chat UI
```

(https://developers.cloudflare.com/agents/getting-started/build-a-chat-agent/, https://developers.cloudflare.com/agents/api-reference/http-sse/)

## 9. AI model calls

### Workers AI (no API key)

### `McpAgent`

```typescript
import { McpAgent } from "agents/mcp";
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { z } from "zod";

export class MyMCP extends McpAgent<Env, State, Props> {
  server = new McpServer({ name: "my-server", version: "1.0.0" });

  async init() {
    this.server.tool("add",
      "Add two numbers",
      { a: z.number(), b: z.number() },
      async ({ a, b }) => ({ content: [{ type: "text", text: String(a + b) }] })
    );
    this.server.resource("counter", "mcp://resource/counter", (uri) => ({
      contents: [{ uri: uri.href, text: String(this.state.counter) }],
    }));
  }
}

// mount as a Worker
export default MyMCP.serve("/mcp", { jurisdiction: "eu" /* or "fedramp" */ });

// or with OAuth
export default new OAuthProvider({
  apiHandlers: { "/mcp": MyMCP.serve("/mcp") },
  // ...
});
```

`McpAgent` exposes the same surface as `Agent`: `state`, `initialState`, `setState`, `onStateChanged`, `sql`, plus `props` (OAuth user identity + tokens) and `elicitInput()` for HITL.

### `createMcpHandler` (stateless)

```typescript
import { createMcpHandler } from "agents/mcp";
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";

function createServer() {
  const s = new McpServer({ name: "Hello", version: "1.0.0" });
  s.tool("hi", {}, async () => ({ content: [{ type: "text", text: "hi" }] }));
  return s;
}

export default {
  fetch: async (request, env, ctx) => {
    const server = createServer();    // MUST create per-request
    return createMcpHandler(server, {
      route: "/mcp",
      enableJsonResponse: false,
      sessionIdGenerator: () => crypto.randomUUID(),
      onsessioninitialized: (sid) => {},
      corsOptions: { /* ... */ },
      storage: env.MCP_KV,             // optional, persists session state
    })(request, env, ctx);
  },
};
```

**Critical security note:** MCP SDK ≥ 1.26.0 requires a fresh `McpServer` instance per request in stateless mode. A shared global instance fails after the first request and would leak responses across clients (CVE) (https://developers.cloudflare.com/agents/api-reference/mcp-handler-api/).

## 15. Gotchas / footguns (collected)

Every "Note" or "Warning" found across the crawled pages, deduplicated.

**Decorators**
- Do **not** set `experimentalDecorators: true`. SDK uses TC39 standard decorators; legacy flag silently breaks `@callable()`. Extend `agents/tsconfig`. (https://developers.cloudflare.com/agents/api-reference/configuration/, https://developers.cloudflare.com/agents/api-reference/callable-methods/, https://developers.cloudflare.com/agents/getting-started/quick-start/)
- Set `"target": "ES2021"` in tsconfig for proper TC39 transpilation through Vite/esbuild. (https://developers.cloudflare.com/agents/getting-started/quick-start/)
- `agents/vite` plugin is required for decorator support (and safe to include even if unused). (https://developers.cloudflare.com/agents/api-reference/configuration/)

**Wrangler / migrations**
- `nodejs_compat` is mandatory. (https://developers.cloudflare.com/agents/api-reference/configuration/)
- Migration tags are append-only — never modify an existing migration; tags must be unique and sequential. (https://developers.cloudflare.com/agents/api-reference/configuration/)
- `deleted_classes` permanently deletes all data. (https://developers.cloudflare.com/agents/api-reference/configuration/)
- Wrangler `vars` values must be strings — parse numbers/bools in code. (https://developers.cloudflare.com/agents/api-reference/configuration/)
- Renaming a workflow binding: call `migrateWorkflowBinding()` to preserve tracking records. (https://developers.cloudflare.com/agents/api-reference/run-workflows/)

**State**
- State must be JSON-serializable; functions, Dates, Maps, Sets do not serialize. Use ISO strings. (https://developers.cloudflare.com/agents/api-reference/store-and-sync-state/)
- Always use `setState()`, not direct mutation, or persistence + broadcast won't fire. (https://developers.cloudflare.com/agents/getting-started/quick-start/)
- `validateStateChange()` is the validator. `onStateChanged()` is **not** for validation — it is a notification hook and must not block broadcasts. (https://developers.cloudflare.com/agents/api-reference/store-and-sync-state/)
- Per-instance: state never crosses instances. (https://developers.cloudflare.com/agents/api-reference/store-and-sync-state/)

**Hibernation**
- In-memory class fields, timers, in-flight promises, and closures are lost. Persist to `this.state` or SQLite. (https://developers.cloudflare.com/agents/api-reference/websockets/, https://developers.cloudflare.com/agents/concepts/long-running-agents/)
- For long work, use `keepAliveWhile` or delegate to a Workflow and hibernate; do not spin in `setInterval`. (https://developers.cloudflare.com/agents/concepts/long-running-agents/)

**Routing**
- Do not modify the response from `routeAgentRequest()` for WebSocket upgrades. (https://developers.cloudflare.com/agents/api-reference/calling-agents/)
- Sensitive instance names: set `static options = { sendIdentityOnConnect: false }` to avoid leaking session/user IDs to clients. With this off, `agent.identified` stays false and `agent.ready` never resolves. (https://developers.cloudflare.com/agents/api-reference/routing/)
- `basePath`: identity may flip on reconnect (e.g. expired session) — handle `onIdentityChange` or you'll get warning logs. (https://developers.cloudflare.com/agents/api-reference/routing/)

**Scheduling**
- Scheduling a non-existent method **throws at execution**. (https://developers.cloudflare.com/agents/api-reference/schedule-tasks/)
- Cron is idempotent by default; delayed/Date schedules are not — opt in via `idempotent: true`, especially when seeding from `onStart()`. (https://developers.cloudflare.com/agents/api-reference/schedule-tasks/)
- `scheduleEvery` skips overlapping executions if the previous run hasn't finished. (https://developers.cloudflare.com/agents/api-reference/schedule-tasks/)
- Sub-agents cannot schedule (or `keepAlive`); have the parent dispatch. (https://developers.cloudflare.com/agents/api-reference/sub-agents/)

**Retries / queues**
- No dead-letter queue: tasks are dropped after `maxAttempts`. (https://developers.cloudflare.com/agents/api-reference/retries/)
- Head-of-line blocking on the queue: a slow retry blocks subsequent items. (https://developers.cloudflare.com/agents/api-reference/retries/)
- No circuit breaker — persistent failures keep retrying per task. (https://developers.cloudflare.com/agents/api-reference/retries/)
- `shouldRetry` exists on `this.retry()` only; not on `schedule()` or `queue()`. (https://developers.cloudflare.com/agents/api-reference/retries/)
- Long retry backoffs keep the DO awake — for long delays use scheduling instead. (https://developers.cloudflare.com/agents/api-reference/retries/)
- Only retry idempotent operations from clients on disconnect — automatic re-issue duplicates non-idempotent calls. (https://developers.cloudflare.com/agents/api-reference/callable-methods/)

**Workflows from agents**
- Workflows cannot open WebSockets; broadcast through the Agent. (https://developers.cloudflare.com/agents/api-reference/run-workflows/)
- Non-`step` Agent calls may execute multiple times on retry — treat as best-effort. (https://developers.cloudflare.com/agents/api-reference/run-workflows/, https://developers.cloudflare.com/agents/concepts/workflows/)

**Chat / AIChatAgent**
- Always forward `abortSignal` into `streamText`, or LLM keeps generating after cancel. (https://developers.cloudflare.com/agents/api-reference/chat-agents/)
- `AbortSignal` cannot cross DO RPC boundaries — construct controllers inside the DO. (https://developers.cloudflare.com/agents/api-reference/chat-agents/)
- Partial chunks persisted before an abort remain in SQLite. (https://developers.cloudflare.com/agents/api-reference/chat-agents/)

**Sub-agents**
- Child class must extend `Agent` and be exported from the worker entry point with the **exact** class name (no aliased exports). (https://developers.cloudflare.com/agents/api-reference/sub-agents/)
- Children are auto-discovered via `ctx.exports`; only the parent needs a DO binding. (https://developers.cloudflare.com/agents/api-reference/sub-agents/)
- `abortSubAgent`/`deleteSubAgent` cascade through the child's own sub-agents. (https://developers.cloudflare.com/agents/api-reference/sub-agents/)

**Durable execution**
- Closures inside `runFiber` are **not** serialized — recovery logic must live in `onFiberRecovered`. (https://developers.cloudflare.com/agents/api-reference/durable-execution/)
- If `onFiberRecovered` throws, the row is still deleted with no automatic retries. (https://developers.cloudflare.com/agents/api-reference/durable-execution/)
- `stash()` fully replaces the previous snapshot; data must be JSON-serializable. (https://developers.cloudflare.com/agents/api-reference/durable-execution/)

**Readonly connections**
- Readonly check fires inside `setState()`, **not** at the top of a callable — side effects in the callable still run. Order code so you call `setState()` first if side effects must be conditional. (https://developers.cloudflare.com/agents/api-reference/readonly-connections/)

**MCP**
- MCP SDK ≥ 1.26.0 stateless servers must create a fresh `McpServer` per request — sharing a global instance fails after the first request and would leak responses across clients (CVE). (https://developers.cloudflare.com/agents/api-reference/mcp-handler-api/)
- McpAgent state resets after the session ends — the architecture does not preserve state across reconnections. (https://developers.cloudflare.com/agents/api-reference/mcp-agent-api/)
- Elicitation requires client support — not all MCP clients implement it. (https://developers.cloudflare.com/agents/api-reference/mcp-agent-api/)
- MCP transport: SSE is **deprecated**; use Streamable HTTP. (https://developers.cloudflare.com/agents/model-context-protocol/transport/)
- MCP client URL validation rejects private IPs, cloud metadata endpoints, link-local/loopback (except localhost). (https://developers.cloudflare.com/agents/api-reference/mcp-client-api/)
- MCP server design: do not wrap your full API; deploy focused, narrowly-scoped servers. (https://developers.cloudflare.com/agents/model-context-protocol/)

**Codemode**
- Experimental — breaking changes likely. (https://developers.cloudflare.com/agents/api-reference/codemode/)
- `needsApproval` is **not honored** — approval-required tools execute immediately. (https://developers.cloudflare.com/agents/api-reference/codemode/)
- Network blocked by default at the executor level. (https://developers.cloudflare.com/agents/api-reference/codemode/)

**Browser Run**
- Each `browser_execute` opens a fresh session — no authenticated/cookied state across calls. (https://developers.cloudflare.com/agents/api-reference/browse-the-web/)
- External network blocked at runtime. (https://developers.cloudflare.com/agents/api-reference/browse-the-web/)
- Responses truncated to ~6,000 tokens. (https://developers.cloudflare.com/agents/api-reference/browse-the-web/)
- Requires `@cloudflare/codemode` peer dep + `LOADER` (Worker Loader) binding in addition to `BROWSER`. (https://developers.cloudflare.com/agents/api-reference/browse-the-web/)

**Auth (web app, not MCP)**
- Always verify webhook signatures (HMAC-SHA256). (https://developers.cloudflare.com/agents/guides/webhooks/)
- For OAuth-protected MCP endpoints: use `__Host-` cookies (Secure, Path=/, no Domain) for CSRF; CSP with `default-src 'none'`, `frame-ancestors 'none'`; HMAC-sign approved-clients registry; reject `javascript:`, `data:`, `file:` URI schemes from client metadata. (https://developers.cloudflare.com/agents/guides/securing-mcp-server/)

**Email**
- Sign reply emails with a secret to enable secure-reply routing — header forgery is otherwise possible on inbound replies. (https://developers.cloudflare.com/agents/api-reference/email/)

**Observability**
- Diagnostics channels are silent by default (zero overhead unless subscribed). In production, all messages auto-forward to Tail Workers. Channels: `agents:state`, `agents:rpc`, `agents:message`, `agents:schedule`, `agents:lifecycle`, `agents:workflow`, `agents:mcp`, `agents:email`. (https://developers.cloudflare.com/agents/api-reference/observability/)

**RPC method serialization**
- All `@callable()` args/returns must be JSON-serializable; functions, Dates, Maps, Sets do not survive. (https://developers.cloudflare.com/agents/api-reference/callable-methods/)
- For server-to-server calls (Worker → Agent or Agent → Agent), prefer `getAgentByName().method()` over `@callable()` (no decorator needed; standard DO RPC). (https://developers.cloudflare.com/agents/api-reference/callable-methods/)
