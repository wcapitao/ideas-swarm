MVP — Combines + critiques

AI Ideator

Generating novel ideas by combining arXiv papers through Combinatorial Creativity

Minimum Viable Product

MVP Architecture

One durable object (IdeatorAgent): distant paper pairs, a combiner batch (ideas + self-scores), then parallel adversarial evaluation — preview of critique, not the full evaluator council.

Step by Step

How the MVP Works

From user prompt to markdown idea cards with self-scores and an evaluator block — assembled on the Worker and streamed once for chat persistence.

1

User Describes a Problem

The user types a topic or problem into the chat interface. Example: "How can we make LLM agents more creative?" — the agent receives this via WebSocket.

2

Select Maximally Distant Paper Pairs

The agent computes Jaccard distance across all papers using their tags, categories, and domains. It greedily selects 4 pairs with maximum distance — forcing cross-domain combinations. No paper appears in more than 2 pairs.

3

Combiner Batch: 4 Parallel Calls

Each pair is sent to DeepSeek via AI Gateway in parallel. The prompt includes the user's problem plus each paper's topic, tags, novelty claims, open problems, and applicability. Every response is validated with Zod into a structured IdeaCard — combined narrative, novelty rationale, applications, and generator self-scores (1–10).

4

Parallel adversarial evaluation

Each validated card triggers a second DeepSeek call through the same AI Gateway (evaluator.ts). A medical-evidence persona returns evidence gaps, safety flags, adjusted scores (versus the generator's self-scores), and a one-line verdict. Still one Durable Object — this is not the multi-judge evaluator council planned for Phase 4.

5

Markdown report, then stream

The Worker builds one markdown document: every idea shows sources, narrative, generator scores, and (when parsing succeeds) an Evaluation section with verdict, adjusted-score bars with ↑/↓, gaps, and flags. That document is echoed through streamText so the chat agent persists a single assistant turn — deterministic layout, streamed delivery.

The Full Pipeline

Full Vision

The complete system: from arXiv papers to evaluated, traceable novel ideas through concept extraction and recombination.

1

Ingest arXiv Papers

arXiv papers are ingested, summarized, and indexed into the knowledge base. Each paper gets a wiki article with key-concept extraction, tiering, and full citations. The KB spans combinatorial creativity, computational creativity, LLM ideation, and agentic patterns.

2

Extract & Retrieve Concepts

Atomic concepts are extracted from arXiv papers and embedded into a searchable index. The retrieval layer uses distance-controlled sampling — not just nearest neighbors, but deliberately pulling concepts from distant domains (the Mednick/remote-associates mechanism) to maximize creative potential.

3

Combinatorial Blending

This is the core innovation. The AI engine combines retrieved concepts using conceptual blending — a cognitive science technique that creates new ideas by finding emergent structure between unrelated concepts. It generates multiple novel ideas, each fully traceable to its source arXiv papers and extracted concepts.

4

Evaluate & Rank

A council of specialized AI evaluators scores each idea on: novelty (is this a known connection or genuinely new?), utility (is it useful and actionable?), surprise (does it violate expectations productively?), and feasibility (can it be built?). Only ideas that pass all evaluators are presented.

Note — AI Ideator is a research prototype. Generated ideas are starting points for exploration, not finished solutions. Every idea traces back to its source arXiv papers — verify claims against the original research.
