Abstract
Consider the problem of merging inside a garbled circuit (GC) two arrays of w-bit elements, yielding
a single length-n array. This garbled merge problem is core to garbled random access memory (GRAM),
a technique that enables efficient garbling of general-purpose programs. We present a novel symmetric-
key-based garbled merge that achieves a garbling size of (w+1)·n·λ bits, providing both asymptotic and
concrete improvements over the state of the art. By applying our garbled merge, we obtain a symmetric-
key GRAM of size O(n lg3 n · λ) · ω(1) for a word RAM program that manipulates words of size Θ(lg n)
bits and halts within n steps, improving over the previous best result (Heath et al., CRYPTO’23) by an
O(lg lg n) factor. This communication cost was previously only achieved under the public-key-style DDH
assumption (Gu et al., CRYPTO’25). We implement our construction, and our evaluation shows that
our garbled merge reduces the communication cost over the DDH-based merge by about 3×.
Keywords: Garbled Circuits, Garbled RAM, Secure Multiparty Computation

Introduction
Secure two-party and multi-party computation (2PC and MPC) allow mutually untrusting parties to jointly
compute a function over their private inputs, while revealing nothing beyond the function’s output. Yao’s
Garbled Circuit (GC) is a fundamental primitive in cryptography, central to 2PC/MPC [Yao86]. In GC,
one party, called the garbler (G), constructs a garbling of a program P, denoted ˜P. Traditionally, P is a
Boolean circuit. The garbler G conveys to the evaluator E the garbled program ˜P, along with encryptions
of the parties’ inputs. E then executes the garbled program ˜P gate-by-gate: at each gate, E uses encodings
of wire values to decrypt subsequent encodings of wire values, ultimately leading to E holding an encoding
of program output. G and E then jointly decrypt that output and recover P(x). GC enables 2PC/MPC
protocols that use only a constant number of communication rounds, making the technique a good choice
when network latency is high.
Improving the asymptotic and concrete cost of GC is critical for making constant-round 2PC/MPC
more practical. The primary goal is reducing communication complexity, dictated by the size of the garbled
program ˜P. This is because communication is often the bottleneck in secure computation, particularly in
bandwidth-constrained settings. Trading increased local computation for reduced communication is gener-
ally favorable, as modern processors can perform millions of cryptographic operations per second. Space
complexity, though often overlooked, is equally important: parties may face prohibitive memory overhead
from large garbled circuits and wire encodings. Notably, reducing the garbled program size does not nec-
essarily reduce overall space, as advanced GC techniques (including ours) can produce wire encodings that
asymptotically exceed the garbled program itself.
Garbled RAM
While much of the GC literature focuses on secure evaluation of Boolean circuits, it is also
possible to garble other computational primitives. One of the most important special-case garbling problems
is that of garbled random access memory (GRAM) [LO13]. GRAM is essential for garbling general-purpose
programs. Without efficient RAM access, off-the-shelf programs would incur prohibitive overhead when stor-
ing or manipulating large amounts of data in non-trivial data structures. Because of its importance, GRAM
has been studied extensively, see e.g. [LO13, GHL+14, GLOS15, GLO15, HKO22, PLS23, HKO23, GTRS25].
Roughly speaking, GRAM is achieved by implementing Oblivious RAM inside the garbled program [GO96].
Its efficiency is typically measured by the size of the garbling of an n-step RAM program manipulating w-bit
words.

The challenge of GRAM
The crucial technical challenge1 of GRAM is one of routing. To achieve random
access memory, array elements must be routed from one part of the garbled program (where the memory
element is stored) to another (where the element is used). Routing in GC is difficult even when the evaluator
E is told in cleartext how all data should move. The source of this difficulty is that the garbler G cannot
learn these routing details without breaking GC’s central constant-round property. Indeed, routing decisions
might not be computable until runtime, when the garbler G is not available.
In standard GC techniques (including ours), each wire value is encoded by a particular ciphertext.
Therefore, if we consider an array x, we can encode each of its bits xi by a distinct ciphertext, which we
denote by [xi]. Now, suppose G would like to prepare a garbled gate that takes as input some particular
wire xi where the array index i is decided at runtime.
To correctly garble a particular gate, G must know the relevant ciphertexts in advance. Since array
index i is revealed only at runtime, G cannot anticipate which ciphertext [xi] will be needed. As a result,
G cannot directly garble the gate ahead of time, despite the fact that E can easily access the correct
ciphertext [xi] once i is revealed. Indeed, known techniques for GRAM include additional structure that
routes memory elements to their eventual destinations. Such routing structures can be achieved by simple
Boolean gates. However, this incurs additional garbled gates, and hence additional cost. Known Boolean-
circuit-based sorting networks incur polylogarithmic blow-up in total cost that we would like to avoid. Our
main contribution is a routing structure that requires asymptotically less communication.
In the GRAM literature, the routing challenge is called the language translation problem. Each wire has
a specific language – the possible ciphertexts that can appear on that wire at runtime. To place a garbled
value onto a particular wire, E must translate her encoding to the language of that wire.

Technical Overview
Our technical observations are two-fold:
• The stable merge of two arrays can be encoded in the execution of a particular finite state automaton
(FSA).
• The transition behavior of our FSA can be encoded as a particularly garbling-friendly system of con-
straints.
In particular, prior work [Hea24] showed that certain systems of constraints—namely, those formalized in a
model of computation called the switch system model—can be efficiently garbled. We encode our FSA as a
garbled switch system. This switch system incurs particularly low communication cost, leading to our main
result.
This section describes our techniques at a high level. We start by reviewing the switch system model
and highlighting its advantages, in particular its ability to efficiently compute on one-hot vectors. Then, we
describe our merge FSA and explain how we encode its behavior as a switch system.
2.1
Switch System Review
Since our approach is based on switch systems [Hea24], we begin by reviewing the computational model
to establish common background knowledge. We focus on the model’s main two gates: switches and joins.
Section 4 provides further details on the switch system model.
We note that the switch system model
was proposed in the context of garbled arithmetic operations; our work shows it is also highly relevant to
non-trivial problems in garbled data structures.
In general, any computation can be understood as solving a system of constraints (the program) over
program variables, including inputs, outputs, and intermediate variables. From this perspective, Boolean
circuits can be understood as a method for writing restricted forms of constraint systems. The switch system
model offers a more flexible collection of primitives for expressing constraints. While Boolean circuits express
constraints that must be solved from left to right (i.e., from inputs to outputs), switch systems allow greater
flexibility in that their gates can express constraints in the true sense, where an evaluator may solve for
variables in whichever order is convenient to the current execution. We leverage this additional expressivity
in our handling of garbled merge.
More precisely, a switch system is defined over a set of variables or wires; its gates specify logical con-
straints on those variables. The central gate in the switch system model is called a switch: A switch (

i.e., x := y ⇐ctrl

Looking ahead, garbled evaluation of a switch requires the evaluator E to know the control bit ctrl in
cleartext. This is because E will act conditionally depending on ctrl. If ctrl holds zero, then E uses a hash
function to “translate” between the two data wires; else, E does nothing. This introduces two points: (1)
all cleartext switch controls must be computable by E, and (2) the cleartext controls must reveal no useful
information about private inputs. Our garbled merge incorporates many switches, but all control values can
be inferred from the merge’s input control string s, and s can be simulated due to the garbled RAM’s use
of ORAM techniques.
A join gate (x ▷◁y) unconditionally enforces the equivalence of its two inputs x and y. From an execution
perspective, if one wire is defined while the other is undefined, the join gate sets the undefined wire to the
defined value. From the perspective of the constraint system, this ensures that the value of either wire can
always be inferred from the other as soon as one becomes defined.
Crucially, join gates are the only gates that incur significant communication when garbled. Switches, for
instance, require no communication. For the unfamiliar reader, it may be unclear why joins are needed, since
they simply express an equality constraint. Join gates are necessary due to the syntactic rules enforced by
the switch system model. Suppose, for instance, we would like to instantiate two switches whose output data
wires are, in fact, the same wire. Such an arrangement could be used to implement the following conditional
assignment:
if ctrl = 0 then z := x else z := y
The switch system model explicitly disallows expressing this conditional as two lone switches. Indeed, each
switch outputs a distinct, fresh wire. That is, there is a difference in how wires are treated when the switch
system is specified (wires are created by specifying gates from left to right) from how they are evaluated
(gates specify multidirectional constraints). Looking ahead, this duality reflects the fact that the garbler
handles switch systems in one way (he garbles, left to right), and the evaluator handles them in another (she
evaluates, in whichever order computes the output).
If we wish to state that two wires are “the same”, we can do so by means of a join. The insistence on
using joins may seem superfluous, but it is motivated by the cost of GC techniques. For instance, known
symmetric-key GC schemes require at least one ciphertext (i.e., λ bits) of communication to implement the
above conditional assignment. The switch system captures this in the sense that expressing the conditional
assignment requires a single join. Each garbled join gate incurs λ bits of communication.
Switch systems also support XOR gates. These are standard fan-in-two gates, except that they act as
constraints, so once any two connected wires become solved, the evaluator can solve for the third. Garbled
XOR gates are communication free [KS08].
Finally, since switch systems require that the evaluator learn switch controls, we need a mechanism by
which to reveal values. We consider a simple reveal gate which achieves this task. Again, for security one
must prove a particular switch system has the property that all revealed values provide no useful information
about the input – the switch system should be oblivious [Hea24].
We note that our handling of revealing bits is slightly different from that of [Hea24]. While we consider
reveal gates, [Hea24] instead explicitly reveals the control bit of each switch. This is a small change, but
it allows us to more precisely account for communication cost, since each revealed bit incurs one bit of
communication. As it turns out, our merge construction requires no reveal gates, since all switch controls
can be inferred from the merge’s control string s. Hence the only communication in our merge comes from
its join gates.

For example, (q0, q1, q2, q3) = (0, 1, 0, 0) is a one-hot vector.

that in the scaling system, wire r0 is the output of a switch controlled by q0. To garble this switch, G derives
a key Kr0 from Kq0:

once the evaluator E obtains Kq0, and once she learns that q0 = 0, she can derive the corresponding key

2.2
Garbled Merge as a Finite State Automaton
Recall that our main technical observation is to encode the partially-private merge of two arrays in the
execution of a finite state automaton (FSA). As we will see in detail soon, state transitions in this FSA
correspond to reading particular elements from either x0 or x1. From here, we observe that the execution
of this FSA can be encoded by a particular switch system C, made efficient by carefully leveraging one-hot
scaling operations. Because of this care, most gates in C are “free”, in the sense that, when garbled, their
garbling requires no communication. Indeed, C has only O(nw) join gates, and these are C’s only gates that
incur communication.
We first demonstrate how to encode a merge in the execution of an FSA, and then how to encode the
execution of that FSA as switch-system-based constraints.

3.2
Garbling Schemes
Our technique can be formalized by means of a garbling scheme [BHR12].
Roughly, a garbling scheme
is a tuple of procedures that specify ow to garble a program and how to evaluate that garbled program.
The crucial properties are correctness, which states that the garbled program outputs the same value as
the cleartext program (with overwhelming probability) and privacy, which states that the garbled program
can be simulated, given only the program and its output (and not the input). Appendix A provides a full
definition.
3.3
Cryptographic Assumption
Our garbling techniques are secure assuming the existence of a hash function satisfying a concrete property
called circular correlation robustness (CCR) [CKKZ12, ZRE15]. It is typical to instantiate a CCR hash via
fixed-key AES, and this instantiation is relatively well studied; see e.g. [BHKR13, GKWY20]. Appendix B
provides a formal definition.
3.4
Tri-State GRAM; How Our Merge Plugs In
Our main contribution is an improved GRAM construction, based on our garbled merge primitive. Specifi-
cally, we build upon the Tri-State GRAM of [HKO23]. We replace one of its core components—circuit-based
stacks—with our more efficient garbled merge.
While the [HKO23] GRAM is formalized as a so-called tri-state circuit (TSC), not as a switch system,
the switch system model is formally more expressive that the TSC model; [Hea24] provides a proof. Every
(total) TSC is automatically and directly implementable as a switch system. Indeed, switch systems are a
natural generalization of TSCs. Hence, the [HKO23] GRAM can be understood as a switch system, allowing
clean interoperability of our techniques.
This section reviews the Tri-State GRAM, with a focus on its stacks and their cost. We then explain
why the circuit-based stacks can be correctly replaced by our garbled merge. Finally, we analyze the cost of
our garbled merge in the new GRAM construction.

Stacks in Tri-State GRAM
As shown in Figure 5, Tri-State GRAM [HKO23] is structured as a binary
tree. Each internal node maintains two circuit-based stacks [ZE13], one for each child. These stacks are used
to solve the runtime routing problem, allowing parent nodes and child nodes to dynamically communicate.
Each stack supports a conditional pop (cpop) operation: given a control bit p, the stack returns its top
element if p = 1, or an all-zero string if p = 0. For each access, the parent uses the data’s traversal path
in the tree to determine which stack receives a control bit of 1, while the other receives 0. The children
manage the stack contents and return values accordingly. For a stack of size t with word size w, the [ZE13]
implementation incurs O(w · t lg t) joins.
Replacing Stacks with Garbled Merge
Each pair of stacks in Tri-State GRAM can be replaced by one
instance of our garbled merge primitive. Consider two stacks of size t that each supports 2t cpop operations
(of which t have control bit 0 and t have control bit 1). Let x0 be the t w-bit elements of one stack, let
x1 be the t w-bit elements of the other stack, and let s ∈{0, 1}2t with Hamming weight n be the sequence
of control bits. Suppose we call cpop 2t times on each stack (the x0 stack using negated control string ¬s,
the x1 stack using s). If we element wise XOR the two resulting arrays of length 2t, we obtain precisely
the merge of x0 and x1 as prescribed by control bits s. By Theorem 2, to replace two stacks of size t, our
garbled merge approach incurs (w + 1) · (2t) (or O(w · t)) joins. This is a factor-lg t improvement over the
stack construction in Tri-State GRAM.
While our garbled merge requires that the evaluator learn the control string s, revealing this value does not
compromise obliviousness of GRAM. Indeed, garbled merge only leaks the control string s, which corresponds
precisely to the leakage of cpop operations in the original Tri-State GRAM. Each bit in s indicates which
child node is active and thus reveals the traversal path down the tree. However, this path information is
permitted to leak in the underlying Circuit ORAM construction [WCS15]. Even with the path revealed,
Circuit ORAM ensures that the actual memory access pattern remains hidden. Therefore, the obliviousness
of the GRAM construction is preserved.
Stacks support online behavior by outputting one element at a time during cpop operations. We empha-
size that our garbled merge primitive preserves this property. As previously discussed, our switch-system-
based FSA is online: it allows merging to proceed as control bits arrive, and partial outputs can be read
immediately. Thus, garbled merge supports online output in a manner similar to that of stacks.

are 2i nodes. A naive approach would allocate a stack of size O(n/2i) for each node to support O(n/2i)
accesses.

Space Complexity.
Recall that one of our main goals is to improve space complexity. The space com-
plexity of the garbler (resp. evaluator) can be upper bounded by summing two quantities:
• The size of the garbled program.
• The number of wires that must be kept in memory to garble (resp. evaluate) the remaining wires,
multiplied into λ. We wish to minimize this quantity without recomputing the encoding of any wire.
In Boolean GC, these two quantities are typically related by a constant factor, as the number of wires is
proportional to the number of gates, and AND gates typically incur O(λ) bits of communication. However,
in our construction this is not the case. Our switch system has O(nmw) wires, but its garbling has size only
O((n + m)wλ) bits.
Despite the fact that our construction has a large number of wires, it can be garbled (resp. evaluated)
in only O((n + m)wλ) bits of space. To show this, and since the garbling of our program fits within this
asymptotic budget, it suffices to show that the garbler (resp. evaluator) needs only store O((n + m)w) wire
encodings.
Let us first discuss the evaluator’s space. We can first examine the space needed to propagate indicator
variables. Invariant 1 clearly implies that it suffices to keep track of the encodings of at most O(n + m)
indicators. Indeed, because the FSA has a grid structure, it suffices to keep the encoding of the single active
state qi,j as well as those indicators exactly one step above and/or exactly one step to the left of qi,j. There
are only O(n + m) such indicators.
The evaluator must also track values readb
i,j (Equations (5) to (7)). These are easy to handle in O((n +
m)wλ) bits of space.
In particular, notice that Equations (6) and (7) involve XOR sums of variables
readb
i,j, and these are the only ways that these variables are used. Thus, rather than storing an encoding
of each variable readb
i,j in local memory, the evaluator instead keeps track of O(nm) accumulators (each
corresponding to a w-bit word) corresponding to partial sums in those equations. As soon as the evaluator
computes a new encoding readb
i,j, she eagerly XORs into the appropriate accumulators, then discards the
encoding.
The garbler’s handling is easier than the evaluator’s.
In short, the garbler can proceed row by row
through the FSA, using indicator encodings on row i to derive indicator encodings and read values on row
i+1, as described by the switch system constraints. Once he has garbled row i+1, he discards the encoding
of row i. To handle encodings of variables readb
i,j, the garbler follows the evaluator’s strategy and maintains
accumulators rather than keeping all such variables in memory. Thus, he also runs in O((n + m)wλ) space.

5.2
Randomized Merge
One problem of our merge construction is that it imposes high computational cost. Suppose that our two
merged arrays x0, x1 are each of length n. Our construction presented so far requires that G and E each
invoke a CCR hash function O(w · n2) times, which is expensive. However, for particular use-cases – namely
those where the control bits are randomly distributed – we can do far better, invoking the hash function
only O(w · n ·
√
n · λ) times.
Suppose that the control bit string s is sampled uniformly, subject to the constraint that it has Hamming
weight n. Note that the Tri-State GRAM’s usage of merge has this property. By a simple observation about
random walks, we can see that it is only neglibly likely that a random s will cause an FSA execution to
deviate significantly from the main FSA’s main diagonal. That is, it is unlikely the FSA will enter a state
qi,j where |i −j| is a large value.
In particular, an active path through the FSA is likely to deviate only a small distance O(
√
n · λ) from the
FSA’s main diagonal. Because of this, we can greatly improve G’s and E’s compute by simply ignoring those
states that are too far from the main diagonal, and our construction will still correctly implement merge with
overwhelming probability. Appendix C expands this discussion and proves correctness of ignoring states far
from the main diagonal.
5.3
Improved Garbled RAM
By replacing stack-based merging with our garbled merge data structure, we obtain the following key theorem.
Theorem 3 (Switch System RAM). Let P be a RAM program halting in n steps and manipulating w-bit
words. There exists an oblivious switch system C such that for any input x, C(x) = P(x). The system C
has O((w + lg2 n) · n lg n) · ω(1) joins.
Proof. Recall that our GRAM is the Tri-State Circuit GRAM where we replace stacks by our merge compo-
nent. Recall also (from Section 3.4) that every tri-state circuit join gate can be replaced by a switch system
join. The total number of join gates in our Switch System GRAM can be attributed to three sources:
Merge joins. As discussed in Section 3.4, our merge gates contribute a total of O(wn lg n) join gates.
Joins from non-merge gates.
The non-stack joins in Tri-State GRAM arise from the read/write
circuitry needed to obliviously read/rearrange the contents of nodes on root-to-leaf paths of the ORAM
tree. Each RAM operation requires a constant number of such traversals. In the underlying Circuit ORAM
construction, each internal tree node holds O(1) data items, each of size O(w). Since paths are of length
O(lg n), traversing a path consumes O(w lg n) join gates. An additional ω(1) term is incurred from the root
node and leaf nodes, each of which hold a larger number of O(lg n) · ω(1) elements; this is needed to achieve
negligible probability of failure Thus, touching any such node incurs O(wn lg n) · ω(1) joins, which is the
asymptotic bottleneck.
Joins for recursive position maps. As in many ORAM constructions, Tri-State GRAM requires a
recursion step. In particular, it uses a recursively instantiated position map. This position map tracks where
in the ORAM binary tree structure each logical element resides. The number of required levels of recursion
is O(lg n). Note that regardless of the top-level ORAM’s word size, the position map uses a word size of
Θ(lg n) bits. Hence, the total cost of the recursive position map is O(n lg3 n) · ω(1) joins.
Adding the contributions from merge gates, non-merge gates, and recursive position maps gives a total
of O((w + lg2 n) · n lg n) · ω(1) join gates, as claimed.

Evaluation
We implement a prototype of our garbled merge technique and an end-to-end Garbled RAM using our
garbled stack in C++, and use the EMP Toolkit to instantiate the underlying hash function, which is based
on fixed-key AES [WMK16]. All experiments are run on AWS c5ad.8xlarge machine with 64GB RAM. The
security parameter of our scheme is fixed as λ = 128.
We evaluate performance along two dimensions: communication and computation. Communication cost
is measured by the size of garbled programs, while computation cost is measured by the number of AES

Comparison against [GTRS25]
PicoGRAM [GTRS25] also provides a compaction network for merging
using single-instruction-multiple-data (SIMD) gates. We instantiate the DDH group with the elliptic curve
P-256 for merging. Our scheme consistently achieves a lower communication cost than PicoGRAM (see
Figure 9b). Concretely, at n = 211 and w = 128, PicoGRAM’s merging requires approximately 14.8 MB,
whereas ours requires only approximately 4.1 MB, achieving 3× improvement.
We do not plot the number of AES calls made by PicoGRAM’s DDH-based approach because its main
computational cost is group exponentiation. The number of required exponentiations is almost exactly the
same as the AES calls required by the Boolean network. Both techniques use the same compaction network,
involving O(n lg n) gates that conditionally swap two elements. In PicoGRAM, each swap gate incurs w
group exponentiations.
6.2
End-to-end Evaluation.
To quantify the impact of our garbled merging primitive on a full GRAM instance, we conducted an
end-to-end evaluation by replacing only the stack subroutine in the PicoGRAM construction. Following
PicoGRAM’s evaluation methodology, we report the communication cost per RAM access as a function of
N.
Under this identical experimental setup, replacing the compaction network with our garbled stack yields
a consistent constant-factor reduction in end-to-end communication. As shown in Figure 10a, our method
achieves lower communication cost than PicoGRAM across all problem sizes N = 210 to 224, with the curve
lying uniformly below that of PicoGRAM. In addition, our method requires only hash functions, avoiding
the stronger DDH assumption needed by PicoGRAM.
Figure 10b further shows that our approach maintains a significantly lower stack communication ratio
(≈20%) compared to PicoGRAM (≈25-34%). By reducing the stack cost per access by a constant fraction,
our construction shifts the contribution to communication from the stack to the non-stack components of
the GRAM, demonstrating more efficient bandwidth utilization.
