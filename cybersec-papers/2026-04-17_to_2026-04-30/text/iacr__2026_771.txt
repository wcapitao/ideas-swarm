Abstract. This paper presents vector-input keyed hashing modes that
construct collision-resistant pseudorandom functions (CR PRFs) using
a keyed hash function, where a vector refers to a sequence of variable-
length strings. The proposed vector-input keyed hashing modes, VIM1
and VIM2, originate from the intuition that a string-input keyed hash-
ing mode using a compression function results in a vector-input keyed
hashing mode by replacing the compression function with a hash func-
tion. Combined with the recently proposed string-input keyed hashing
modes KHC1 or KHC2, VIM1 and VIM2 are shown to yield CR PRFs
from a compression function satisfying extended collision resistance and
being a secure PRF under related-key attacks. Extended collision resis-
tance means that it is intractable to ﬁnd a distinct input pair whose
output diﬀerence falls within a small set. This paper also introduces a
keyed hashing mode, PVIM, which allows parallel processing of strings
in a vector. However, it requires more calls to the underlying keyed hash
function than VIM1 and VIM2 do. To the best of our knowledge, this is
the ﬁrst proposal of dedicated vector-input CR PRFs.

Introduction
Background. Cryptographic hash functions are useful for various applications
such as message digest, message authentication, and pseudorandom generation.
It is well known that a keyed hash function H that satisﬁes both collision-
resistance and pseudorandomness instantiates computationally hiding and com-
putationally binding string commitment. For message M, one chooses key K
uniformly at random and computes commitment σ ←HK(M). Recently, some
interesting cryptographic schemes were constructed with such collision-resistant

pseudorandom functions (CR PRFs): compactly committing authenticated en-
cryption with associated data (ccAEAD) [11,16] and hash-based post-quantum
EPID signatures [9].
Conventional CR PRFs such as HMAC [3,12] and KMAC [22] are deﬁned
over strings. However, data to be processed have speciﬁc formats or structures in
many practical situations. A recognized method to process such data is to encode
a given datum into a string and feed it to a conventional string-input CR PRF.
An example of such encoding is included in the speciﬁcation of TupleHash [22]
for a sequence of variable-length strings, which is often called a vector. Although
this method always works, the encoding process introduces an eﬃciency loss.
Our Contribution. Our goal is to construct a vector-input CR PRF using a com-
pression function. Our idea is to nest compression-function-based keyed hashing
modes constructing string-input CR PRFs, that is, to replace the compression
function of a keyed hashing mode with a string-input CR PRF using a compres-
sion function. To the best of our knowledge, this is the ﬁrst proposal of dedicated
vector-input CR PRFs.
For the proposed construction, we adopt two string-input keyed hashing
modes KHC1 and KHC2 [19], which are cascades of a compression function.
Precisely, simple nesting is incomplete since it cannot handle the 0-dimensional
vector (the vector with no string). Thus, we ﬁrst present two vector-input keyed
hashing modes VIM1 and VIM2 based on KHC1 and KHC2, respectively. They
construct vector-input keyed hash functions using a string-input keyed hash func-

KHC1 and KHC2 were shown to construct string-input CR PRFs [19]. How-
ever, it is not suﬃcient for constructing vector-input CR PRFs by using them
as a building block of VIM1 and VIM2. Thus, in this paper, KHC1 and KHC2
are shown to satisfy extended collision resistance if the underlying compression
function satisﬁes it. KHC1 and KHC2 are also shown to be a secure PRF under
related-key attacks if the underlying compression function is a secure PRF under
related-key attacks.
Finally, we present a parallelizable vector-input keyed hashing mode, which
constructs CR PRFs by using KHC1 or KHC2. While it enables parallel process-
ing when the strings in a vector are simultaneously available, it requires more
calls to the underlying compression function than VIM1 and VIM2 with KHC1
or KHC2.
Related Work. An iterated hash function consisting of a compression function is
called a Merkle-Damgård hash function [10,25]. Pseudorandom functions were
ﬁrst introduced by Goldreich, Goldwasser, and Micali [15].
KHC1 and KHC2 [19] are inspired by the suﬃx-free-preﬁx-free (SFPF) hash
function construction [1] and KMDP+ [17]. KMDP+, which improves Keyed-
MDP [21], is the most eﬃcient string-input keyed hashing mode of a compression
function to construct a CR PRF. KMDP+ cannot be utilized for VIM1 and VIM2
since it only accepts secret keys shorter than the outputs. Both KHC1 and KHC2
3

are competitive with KMDP+ in terms of the number of calls to the underlying
compression function.
There are many proposals of string-input keyed hashing constructions based
on the Merkle-Damgård construction and satisfying collision resistance and pseu-
dorandomness. They are classiﬁed by their underlying primitives: HMAC [3],
Sandwich construction [28], H2-MAC [29], and AMAC [2] use a cryptographic
hash function, while NMAC [3], EMD [6], Keyed MDP [21], Preﬁx MDP [21],
and KMDP+ [17] use a compression function.
There are several proposals of constructions for a vector-input PRF, which
do not address collision resistance. Rogaway and Shrimpton [27] presented the
ﬁrst vector-input PRF called S2V as a component of their SIV construction
for deterministic authenticated encryption. Minematsu [26] proposed a universal
hash function based on bit rotation and used it to construct a vector-input PRF.
Hirose, Kuwakado, and Yoshida [20] constructed a vector-input PRF based on
the lightweight hash function Lesamnta-LW [18] and MDP [21], employing the
protected counter sum construction [7]. Furuya et al. [13] adopted PMAC2x [24]
using a tweakable block cipher [23] as the baseline scheme and constructed the
vector-input PRF called PMACrx that achieved the beyond-the-birthday-bound
security. Furuya and Minematsu [14] presented a vector-input PRF based on
PMAC [8], which attains optimal eﬃciency with respect to the number of calls
to the underlying block cipher.
Organization. Section 2 gives notations and deﬁnitions. It also introduces KHC1
and KHC2. Section 3 proposes vector-input keyed hashing modes VIM1 and
VIM2. Section 4 shows that both KHC1 and KHC2 construct keyed hash func-
tions satisfying extended collision resistance and being a secure PRF under
related-key attacks. Section 5 conﬁrms that both VIM1 and VIM2 construct
CR PRFs by using KHC1 or KHC2. Section 6 presents a parallelizable vector-
input keyed hashing mode for CR PRFs. Section 7 gives a concluding remark.

2.2
Collision Resistance and Its Extension
Let HF : X →Σ n be a hash function that uses compression function F. An
input pair X and X′ for HF is called colliding if X ̸= X′ and HF (X) = HF (X′).
HF is called collision-resistant if it is intractable to ﬁnd a colliding input pair
for HF .
We extend the notion of collision resistance as follows. For ∆⊂Σ n, an input
pair X and X′ for HF is called ∆-colliding if X ̸= X′ and HF (X)⊕HF (X′) ∈∆.
HF is called ∆-collision-resistant if it is intractable to ﬁnd a ∆-colliding input
pair for HF . {0n}-collision resistance refers to collision resistance.

7
Conclusion
We have proposed vector-input keyed hashing modes, VIM1, VIM2, and PVIM
using a string-input keyed hash function. The sequential modes VIM1 and VIM2
extend conventional string-input keyed hashing modes KHC1 and KHC2 using a
compression function. We have combined VIM1, VIM2, or PVIM with KHC1 or
KHC2 and constructed vector-input CR PRFs by using a compression function
satisfying extended collision resistance and pseudorandomness under related-key
attacks. Though PVIM allows parallel processing, it requires more calls to the
underlying compression function than VIM1 and VIM2.
