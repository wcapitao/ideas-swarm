How to Authenticate a Non-Deterministic Computation
Shift-Hiding Functions, Compressed LWE Sampling, Broadcast Encryption,
and Obfuscation
Damiano Abram∗
Giulio Malavolta†
Lawrence Roy‡
Abstract
We propose a new method to construct homomorphic authentication codes sup-
porting the evaluation of non-deterministic computations, extending the celebrated
homomorphic lattice encodings [Boneh et al., Eurocrypt 2014]. Our approach relies
on the hardness of the decomposed learning with errors problem (LWE), a recently
introduced modification of Regev’s LWE assumption.
We then use this new technical tool to make progress on several open problems in
the literature. Specifically, we obtain:
• A constrained pseudorandom function (PRF), where the evaluation of the PRF
on the master key does not depend on the complexity of the constraint, except
for its circuit depth.
• A way to securely compress and re-expand LWE samples in the plain model.
• An adaptively secure broadcast encryption scheme, with ciphertext and secret
keys growing poly-logarithmically with the size of the encrypted set.
• A pseudorandom obfuscation for all puncturable PRFs, additionally assuming the
existence of sub-exponentially secure indistinguishability obfuscation (iO).
None of the above mentioned primitives was known to exist from lattice assumptions.
As a bonus result, we also obtain a conceptually simple and direct heuristic construction
of iO based on lattice techniques, which is not based on the function encryption-to-
iO paradigm. We provide evidence that this approach can be used to build provably
secure obfuscation for simple functionalities such as sampling lattice preimages using
a hidden trapdoor.

Introduction
Lattice-based cryptography [Ajt96, Reg05] has seen a remarkable success over the past
decade, in part due to its conjectured post-quantum security, in part due to vast number
of new cryptographic primitives enabled by lattices. A central technical toolkit for lattice-
based cryptography is the ability of computing on encrypted data, which is the cornerstone
of primitives such as fully-homomorphic encryption [Gen09, BV11], attribute-based encryp-
tion [GVW13, BGG+14], laconic function evaluation [QWW18], traitor tracing [GKW18],
or succinct arguments [CJJ22], to mention a few. A central technical tool in this area is
2

the notion of lattice encodings, introduced in [BGG+14]. An encoding of a message x with
respect to a matrix M is defined to be
s⊺(M −x⊺⊗G) + e⊺
where G is the gadget matrix [MP12] and e is a low-norm noise term. A useful perspective to
take is to view such an encoding as a message authentication code on the vector x. Indeed it
is not hard to prove that producing an encoding of a different message under the same secret
s leaks non-trivial information about s. On the other hand, the learning with errors (LWE)
[Reg05] assumption postulates that the above encoding is computationally indistinguishable
from uniform, for a random M.
However, the power of such encodings comes from their homomorphic properties. It is
known that, for any function f, one can derive a low-norm matrix Hf,x such that
(s⊺(M −x⊺⊗G) + e⊺) Hf,x ≈s⊺(MKf −f(x)⊺⊗G) + e⊺
where Kf is a (low-norm) matrix that depends only of f. Given that the result is once again
an encoding, by the same argument as before we can think of this process as an authentication
for the computation of a given function f. Indeed, nearly all applications of these encodings,
e.g., [GVW15b, GVW15a, BTVW17, QWW18, Tsa19, PS19, dCP23], implicitly use this
property, one way or another.
A limitation of this authentication code is that the function f needs to be deterministic,
in order for the matrix Kf to be fixed, which is important for applications.
A natural
question is whether one can authenticate randomized computation or, alternatively stated,
authenticate a family of functions {fr}r simultaneously in one encoding. The purpose of this
work is to provide one possible answer to this question.
1.1
Our Results
Our main technical contribution is a new method to authenticate any polynomial-time com-
putable function family {fr}r, where r can be understood as the randomness of the com-
putation and comes from a (possibly) exponentially large domain. We abstract this notion
into a new primitive, which we refer to as non-deterministic encodings, that allows one to
(implicitly) hash lattice encodings of all possible bitstrings r ∈{0, 1}d, homomorphically
compute a function f over the hash, which can be later extended to obtain an encoding of
fr, for any r. Crucially, the computation of the matrix Mf,r used in this encoding can be
split into two parts: first, we derive a matrix Mf that only depends on f, in time propor-
tional to its circuit size; second, we derive Mf,r from Mf in time that depends only on the
length of r and the depth δ of the computation. The circuit commitments of Wee [Wee25,
Section 3.4] achieved similar properties, but the second step in the computation of Mf,r
run in depth proportional to δ · poly(log δ). On the other hand, in our construction, the
extension procedure requires depth log d · poly(log δ). This allows us to homomorphically
run the extension over the non-deterministic encodings in a recursive fashion, enabling new
cryptographic primitives, which prior to our work were unknown from lattices alone.
3

All of our results rely on the hardness of the decomposed LWE problem [AMR25a],
a recent extension of Regev’s LWE problem [Reg05]. In [AMR25a] it is shown that the
hardness of the decomposed LWE problem is implied by the hardness of the succinct LWE
problem [Wee24]. We discuss our applications in more details below and we refer the reader
to Section 1.2 for a more detailed description of the non-deterministic encodings.
Application I: Laconic Shift-Hiding PRFs.
A shift-hiding pseudorandom function
(PRF) [PS18] is a regular PRF except that it has the additional ability to issue shifted keys
for an arbitrary function f so that evaluating the shifted PRF on an input x results into
the output “shifted” by f(x). Furthermore, the shifted key must computationally hide f.
Shift-hiding PRFs are known to imply constrained PRFs [BV15b, BLW17] and correlation-
intractable hash functions [CGH98, LV22].
Using our non-deterministic encodings, we obtain the first laconic shift-hiding PRF,
where the complexity of the (non-shifted) PRF evaluation is independent of the complexity
of shifted function f, except for its circuit depth. This seemingly technical requirement turns
out to be crucial for subsequent applications since it enables the shift function to recursively
evaluate its own PRF, akin to bootstrapping in fully-homomorphic encryption [Gen09].
Theorem 1 (Informal). Assuming the hardness of the decomposed LWE problem, there exists
a laconic shift-hiding PRF for all functions.
Application II: Securely Compressing LWE Samples.
We show a method to securely
compress-and-expand LWE samples with respect to a matrix X. To exemplify the usefulness
of this tool, suppose that Alice holds an LWE secret s and wants to provide an LWE sample
s⊺X + e⊺to Bob. Using our compression algorithm, Alice can send s⊺· Hash(X) + e⊺to Bob,
where the size of Hash(X) is poly-logarithmic in the number of columns of X. Finally, Bob
can locally expand the sample sent by Alice to obtain an LWE sample with respect to X.
For security, we show that if X is random, Alice’s message is indistinguishable from
random, under the hardness of decomposed LWE. This ensures that the secrecy of s is
preserved. This is summarized by the following theorem.
Theorem 2 (Informal). There exist polynomial time algorithms (Hash, Expand) such that,
if h⊺≈s⊺· Hash(X), then Expand(h, X) ≈s⊺· X. Moreover, if X is random, assuming the
hardness of decomposed LWE, h is indistinguishable from random.
Application III: Adaptively-Secure Broadcast Encryption.
Broadcast encryption
[FN94] allows one to broadcast the same message to a set of N participants. The notion
requires the size of the ciphertext to be sublinear (ideally poly-logarithmic) in N. Tradi-
tionally, this notion entailed the presence of a trusted party that pre-distributes keys to the
receivers, but recent work extended it to the distributed settings [BZ14] where each receiver
samples its own key pair.
Arguably, the correct notion of security for broadcast encryption is adaptive security:
The message should be hidden to the eyes of a distinguisher who can select the received set
4

adaptively, after seeing the keys of the receivers. Our main result is the first lattice-based
(distributed) broadcast encryption that is adaptively secure in the plain model.
Theorem 3 (Informal). Assuming the hardness of the decomposed LWE problem, there
exists an adaptively-secure (distributed) broadcast encryption scheme where the size of the
public/private keys and ciphertexts is poly-logarithmic in N.
Note that we achieve the stronger notion of distributed broadcast, but our scheme is
also the first lattice-based (non-distributed) broadcast encryption satisfying the above se-
curity notion in the plain model. Our main technical contribution is an unbounded broad-
cast encryption scheme with essentially optimal parameters satisfying the weaker notion of
semi-static security, then we invoke a recent compiler that generically transforms it into an
adaptively-secure scheme in the standard model [HWW25]. We shall mention that the com-
pilation process from [HWW25] introduces a large common reference string (linear in N),
which is then inherited by our main construction. We leave open the problem of reducing
the size of the common reference string to poly-logarithmic in N, while maintaining adaptive
security.
Prior constructions achieving adaptive security (both in the distributed or traditional
settings) required bilinear pairings [BW06, Wat09] or witness encryption [FWW23] or were
only proven secure in the random oracle model [CHW25]. For context, witness encryption
candidates based on lattices exist [Tsa22, VWW22] but the validity of the underlying as-
sumptions has recently been called into question [B¨UW24, DJM+25]. We also mention some
recent lattice-based schemes satisfying the weaker notion of selective security [WW25], where
the attacker has to commit to the challenge set ahead of time, prior to seeing the keys of the
honest parties.
Application IV: Pseudorandom Obfuscation.
An obfuscation scheme allows one to
make a program unintelligible, without altering its functionality. The standard notion of
security for an obfuscation scheme is indistinguishability obfuscation (iO) [BGI+01], which
requires that the obfuscation of two functionally-equivalent programs must be computation-
ally indistinguishable. A recent work [BDJ+25] proposes a variant of this notion, called
pseudorandom obfuscation for keyed functions.
The security of the primitive postulates
that the obfuscation itself should be computationally indistinguishable from uniform bits,
provided that the obfuscated function is also pseudorandom. However, in the same work
[BDJ+25] the authors showed strong impossibility results for pseudorandom obfuscation for
general functions.
This leaves open the question on what kind of functions admit a pseudorandom obfusca-
tor. We make progress on this question, showing that all puncturable pseudorandom function
admit a pseudorandom obfuscation.
Theorem 4 (Informal). Assuming the sub-exponential hardness of the decomposed LWE
problem and the existence of sub-exponentially secure iO, there exists a pseudorandom ob-
fuscation for all puncturable functions.
5

A puncturable function [SW14] allows one to “puncture out” a point in its domain so
that, even given the ability of evaluating the function on all other points, the output at
the punctured point is computationally indistinguishable from uniform. Conditioned on the
validity of the decomposed LWE assumption, this confirms a conjecture from [BDJ+25].
Application V: A Heuristic Obfuscation Scheme.
As a bonus result, we show how
our laconic shift-hiding PRF immediately yields a fully lattice-based construction of general
purpose obfuscation for any circuit C as follows: Set the shift function to be C(x)−Eval(K, x),
then the shifted evaluation algorithm returns
Eval(K, x) + C(x) −Eval(K, x) = C(x).
Note that the shift function is only well-defined because of the laconic property: The con-
strain depends on the evaluation algorithm, whose complexity potentially depends on the
one of the constrain. The laconic property breaks this circular dependency.
Thus, we obtain a simple and direct obfuscation candidate that is not based on the
functional encryption-to-obfuscation paradigm [AJ15, BV15a], and that is genuinely different
from other lattice-based obfuscation candidates, e.g., [WW21, GP21, BDGM22].
We stress that, contrary to our laconic shift-hiding PRF, we do not know how to reduce
the security of this construction to decomposed LWE (or any other standard assumption)
and we leave a rigorous analysis of this scheme as a fascinating open problem. Nevertheless,
we manage to show that for certain simple pseudorandom functionalities, the approach leads
to provably secure obfuscation.
This is the case, for instance, for circuits that compute
lattice preimages using a hidden trapdoor.

A Heuristic Approach to Obfuscation.
In a laconic shift-hiding function, the evalu-
ation of an unshifted key has lower complexity than the evaluation of a shifted key. Thus,
it is possible, in principle, to evaluate the unshifted PRF inside the shift. This leads to the
following heuristic approach to obfuscation: To obfuscate a circuit C, we first run the setup
algorithm to compute a PRF key K, then we define the function gK to be
gK(x) = C(x) −Eval(K, x)
where Eval is the unshifted evaluation algorithm. The obfuscation will be ˜K
$←−Shift(K, gK).
One can see that evaluating the obfuscated circuit returns the correct result, since
ShiftedEval( ˜K, x) = Eval(K, x) + gK(x)
= Eval(K, x) + C(x) −Eval(K, x)
= C(x).
Of course, the circularity we introduce does not allow us to prove this construction secure, but
we could not disprove it either. If this approach manages to withstand future cryptanalysis,
this approach could provide a more direct way to constructing obfuscation, avoiding the large
computational overheads introduced by recursion in the current iO constructions [BV15a,
AJ15]. In the meantime, we advise caution.
Pseudorandom Obfuscation.
Although the circularity prevents us from proving the
security of our general-purpose iO construction, we shall see how we can circumvent this
barrier, for specific applications. The first is a compiler that turns a regular iO obfuscator
Obf into a pseudorandom one. We recall that a pseudorandom obfuscator guarantees that the
obfuscation of a random PRF is indistinguishable from a random string [BDJ+25]. We show
that our construction satisfies this property if the PRF is puncturable and subexponentially
secure. To obfuscate a circuit C, we first derive ˜g = Obf(1λ, gK), and then output Shift(K, ˜g).
The correctness of the construction follows from the correctness of Obf and of the shift-hiding
function, as shown above. For security, notice that if C(x) computes the evaluation of a
random puncturable PRF, one can apply a puncturing argument to gradually remove all
information about K from ˜g [SW14], at which point pseudorandomness follows directly from
the shift-hiding property.

Broadcast Encryption
In the following we show our main result, namely a construction of adaptively secure (dis-

6.1
Definition
Broadcast encryption [FN94] allows one to broadcast a message to a set of N participants.
The most basic notion [FN94] considers a central key generation algorithm that provides
keys to all participants. Distributed broadcast [BZ14, KMW23] allows each party to sample
their own public-secret key pair. More recently, flexible broadcast [FWW23] further extends
this notion to remove parties identifiers. Here we show a formal definition for the latter.
Syntax.
We recall the syntax of a flexible broadcast scheme.
• Setup(1λ): On input the security parameter 1λ, the setup algorithm returns a master
public key mpk.
• KeyGen(mpk): On input the master public key mpk, the key generation algorithm
returns a public-secret key pair (pk, sk).
• Enc(mpk, pk0, . . . , pkN−1, msg): On input the master public key mpk, a set of public
keys pk0, . . . , pkN−1, and a message msg ∈{0, 1}, the encryption algorithm returns a
ciphertext ct.
• Dec(mpk, k, sk, ct, pk0, . . . , pkN−1): On input the master public key mpk, the index k,
the secret key sk, a ciphertext ct, and the public keys pk0, . . . , pkN−1, the decryption
algorithm returns the message msg.
For correctness, we require that for all k ∈[N] we have
Dec(mpk, k, skk, Enc(mpk, pk0, . . . , pkN−1, msg), pk0, . . . , pkN−1) = msg
where (skk, pkk)
$←−KeyGen(mpk) and mpk
$←−Setup(1λ), except with negligible probability.
For efficiency, we require that the ciphertext size should be bounded by a function that is
(at most) poly-logarithmic in the number of users N.

model. In [HWW25] they show a standard model transformation for the case of distributed
