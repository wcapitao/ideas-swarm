Oriole: Adaptively Secure Partially Non-Interactive Threshold
Signatures from Lattices
Kaijie Jiang1
, Hoeteck Wee2, and Chenzhi Zhu2
1 Institute for Advanced Study, Tsinghua University, Beijing, China
2 NTT Research, California, US
Abstract. We present the first lattice-based, partially non-interactive threshold signature scheme that tolerates
the adaptive corruption of up to T ‚ąí1 signers, where T is the signing threshold. Our construction relies on
the MSIS and MLWE assumptions, and has two rounds, of which only the second is message-dependent. We
substantially improve upon prior adaptively secure lattice-based schemes (CRYPTO ‚Äô24 and EUROCRYPT ‚Äô26),
which require at least two message-dependent rounds. Compared to prior lattice-based partially non-interactive
assumptions (CRYPTO ‚Äô24, S&P ‚Äô25, CRYPTO ‚Äô25), we achieve better communication complexity in addition to
stronger security guarantees.

Introduction
In a T-out-of-N threshold signature [Des88, DF90], a group of N parties hold shares of a signing key so that any
subset of at least T out of N parties can produce a valid signature, while any adversary corrupting fewer than T
of these parties cannot forge new signatures. Threshold signatures play a crucial role in securing decentralized sys-
tems, distributed key management, and fault-tolerant cryptographic protocols. They are widely deployed in practice,
particularly for securing digital wallets and other high-value assets. Ongoing IETF [CKGW22] and NIST [Natnt]
standardization efforts further underscore their growing practical importance. Thanks to a substantial and fruit-
ful body of recent work, we now have efficient threshold signature schemes from pairings [Bol03, BL22, DR24],
RSA [DDFY94, GRJK07, Sho00], pairing-free groups [Mak22, BLT+24, KRT24, BDLR25b, Che25a, CKK+25, BDLR25a,
GCRS25, Che25b, CKK+25, BCL+26], and from lattices [BGGK17, BGG+18, ASY22, GKS24, DKM+24, EKT24, KRT24,
CATZ24, ZT25, BKL+25, dPENP25, PKN+25, BCdP+25, BdCE+25, JTWZ26].
As deployment settings become increasingly adversarial and asynchronous, two properties emerge as central:
minimal online interaction and strong adaptive security against adversaries that may choose which parties to corrupt
as the protocol evolves. In particular, partially non-interactive signing‚ÄĒwhere a single round of message-independent
preprocessing enables signature generation in a single online round‚ÄĒsubstantially improves scalability, latency, and
robustness. However, among existing constructions, only [BL22, DR24, CKK+25, BCL+26, Che26] simultaneously
achieve partial non-interactivity and adaptive security, and they all rely on classical group-based assumptions. Con-
sequently, no known scheme provides these guarantees under post-quantum assumptions, leaving a fundamental
gap in the lattice setting. Addressing this gap is particularly timely, as NIST has explicitly noted the importance of
both post-quantum and adaptive security in potential threshold signature standards.
This work.
In this work, we close this gap with Oriole, the first lattice-based threshold signature scheme that si-
multaneously achieves partially non-interactive signing and strong adaptive security. The security of our scheme is
based on the Module Learning With Errors (MLWE) and Module Short Integer Solution (MSIS) assumptions in the
random oracle model. This is a significant improvement over the only existing adaptively secure lattice-based thresh-
old signature [KRT24, JTWZ26], which requires at least two message-dependent rounds. In Figure 1, we position our
scheme in the broader context of the most closely related schemes.
For 128-bit security, our parameter estimation indicates that Oriole produces signatures of approximately 110KB,
which are even smaller than the 160KB signatures reported for Tweed [JTWZ26], a lattice-based two-round adap-
tively secure scheme. While our signatures are larger than those of prior lattice-based partially non-interactive
schemes achieving only static security [EKT24, ZT25, BKL+25], which obtain sizes in the range of 10‚Äď30KB, our

Our approach.
At a conceptual level, a key obstacle in constructing lattice-based, partially non-interactive thresh-
old signatures with adaptive security lies in the interaction between proof techniques and corruption models. Existing
partially non-interactive schemes [EKT24, CATZ24, ZT25, BKL+25] rely on rewinding-based security proofs, which
are ill-suited for adaptive security. The recent adaptively secure schemes KRT [KRT24] and Tweed [JTWZ26] cir-
cumvent rewinding by adapting techniques originally developed for DL/DDH-based schemes in pairing-free groups
(e.g. Dazzle [Che25a]) to lattices. However, the pairing-free constructions that additionally achieve partial non-
interactivity [CKK+25, BCL+26] rely on stronger assumptions that do not naturally carry over to lattice settings
under standard assumptions.
Motivated by these challenges, we revisit the design space of lattice signatures. The threshold schemes we just
described are based on Fiat‚ÄďShamir-style challenge‚Äďresponse protocols. In contrast, lattice-based hash-and-sign sig-
natures admit non-rewinding proofs [GPV08, MP12, CGM19, YJW23, EEN+24], providing a more promising foun-
dation for adaptive security. However, current threshold hash-and-sign schemes [FNR25, ENP24] achieve only static
security and require three rounds, with a single message-dependent round.
Our approach builds on this alternative foundation. We first show that the hash-and-sign framework can be
adapted to obtain a partially non-interactive threshold scheme with static security and without rewinding. We then
develop new simulation techniques to prove adaptive security.
Concurrent work.
In an independent and concurrent work [Che26], Chen presents a partially non-interactive
threshold signature scheme based on DDH in pairing-free groups; the scheme also tolerates adaptive corruption of
up to T ‚ąí1 signers. The scheme is based on Fiat‚ÄďShamir-style challenge‚Äďresponse protocols.

Comparison with prior schemes.
A crucial distinction from prior lattice-based threshold hash-and-sign schemes
[ENP24, FNR25] is that we derive the salt from {(j, wj)}j‚ąąSS and not w = P
j‚ąąSS wj. This ensures that an adver-
sary cannot substantially bias the distribution of the salt. In particular, an adversary can bias w in our protocol (e.g.,
set w = 0) by choosing its message in the first round as a function of the messages from the honest parties. If the
salt is derived from (¬Ķ, w) as is the case in prior schemes, this would allow the adversary to bias the salt and thus
H1(¬Ķ, salt). Such an attack strategy is avoided in [ENP24] by using distributed key generation to sample w, and
in [FNR25] by using commit-then-open for wi; both of these work-arounds incur at least an extra round of com-
munication. Another recent work [ALLW25] describes a non-interactive threshold variant of GPV signatures using
completely different techniques, but the final signature size grows with the threshold (in particular, the dimensions
of the matrix A underlying the GPV signature grows linearly with T).

Syntax. T-out-of-N partially non-interactive threshold signature is a tuple of algorithms TS = (Setup, Sign1,
Sign2, Combine, Ver) that act as follows.
‚Äď Setup(1őļ) ‚Üí(pk, (sti)i‚ąą[N]): The setup algorithm takes the security parameter as input and outputs a public
key pk and an initial state sti for each signer i.
‚Äď Sign1(i, sti) ‚Üí(pmi, sti): The first-round signing algorithm takes a signer index i ‚ąą[N] and the corresponding
state of signer i as input and outputs an offline nonce pmi and the updated state.
‚Äď Sign2(i, sti, ¬Ķ, SS, M = {(j, pmj)}j‚ąąSS) ‚Üí(psigi, sti): The second-round signing algorithm takes a signer
index i, the corresponding state sti, a message ¬Ķ to be signed, a set of signers SS participated, and a set of first-
round nonces M = {(j, pmj)}j‚ąąSS as input and outputs a partial signature psigi and the updated state sti. The
algorithm may output ‚ä•if the inputs are invalid.
‚Äď Combine(¬Ķ, SS, M, {(i, psigi)}i‚ąąSS) ‚Üísig: The algorithm takes the second-round input (¬Ķ, SS, M) and the
set of partial signatures {(i, psigi)}i‚ąąSS as input and outputs the final signature.
‚Äď Ver(pk, ¬Ķ, sig) ‚Üí1/0: The verification algorithm takes the public key pk, a message ¬Ķ, and a signature sig as
input and outputs 1 (resp. 0) if the signature is valid (resp. invalid).
The honest signing process can be described by the correctness game TS-COR defined in Figure 3. In the first

the Sign2 algorithm and sends the partial signature psigi to the coordinator. Finally, the coordinator runs Combine
